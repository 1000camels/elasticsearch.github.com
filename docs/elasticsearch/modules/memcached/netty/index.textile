h1. Netty memcached Module

p. "Netty":http://www.jboss.org/netty is an excellent asynchronous event-driven network application framework. The netty memcached implementation utilizes it in order to provide high performance, asynchronous based memcached over TCP communication.

h2. Settings

p. Settings for the netty memcached module should be set for within @memcached.netty@. For example, the @port@ setting should be set using @memcached.netty.port@.

|_. Setting |_. Description |
|@port@|A bind port range. Defaults to @11211-11311@.|
|@bind_host@|The bind host. Defaults to the "network level settings":../../node/network.|
|@publish_host@|The publish host. This is the host that other nodes will use to connect to this node. Defaults to the "network level settings":../../node/network.|
|@worker_count@|The number of worker threads to use. Defaults to the number of available processors.|
|@tcp_no_delay@|Enable or disable tcp no delay setting. Defaults to @true@.|
|@tcp_keep_alive@|Enable or disable tcp keep alive. By default not explicitly set.|
|@reuse_address@|Should an address be reused or not. Defaults to @true@.|
|@tcp_send_buffer_size@|The size of the tcp send buffer size (in size setting format). By default not explicitly set.|
|@tcp_receive_buffer_size@|The size of the tcp receive buffer size (in size setting format). By default not explicitly set.|
